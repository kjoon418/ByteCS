# 면접 대비 가치 구현 계획 — 면접 연결 콘텐츠(A) · 연결 문제(D) · 면접 세션(C)

> **문서 성격.** 테스터 피드백 "CS 공부의 목적은 면접 대비인데, 지금의 주관식 문제는 학습보다 단순 암기처럼 느껴진다"를 오너가 수용해 확정한 기획의 실행 계획. 오너 결정(2026-07-21~22, 아래 DI1~DI12)을 전제로 하며, **새 세션이 이 문서만으로 착수 가능**하도록 작성한다.
>
> 도메인 근거: 명세서 기능 1 결정 근거 — *"정의 재생형은 (면접의) 설명하기의 **역방향**이다. … 이 근거는 정의 재생형에 대해서는 성립하지 않는다(정착 근거만으로 선다)."* 테스터가 느낀 괴리는 명세서가 이미 수용했던 트레이드오프가 실사용자에게 노출된 것이다. 이번 작업은 그 회수 — 면접이 요구하는 **이름→설명 방향의 인출**을 서비스에 담는다.

---

## 1. 배경과 목적

- **진단 3가지:**
  1. 명세서 스스로 인정한 방향 불일치 — 면접은 이름을 주고 설명을 요구하는데, 정의 재생형은 설명을 주고 이름을 요구한다(역방향).
  2. 콘텐츠 편중 — 시드 107문제 중 **정의 재생형 91(85%) / 유도형 16(15%)**. 면접 방향과 그나마 정합한 유형이 소수라 체감이 "설명 읽고 용어 맞히기"에 지배된다.
  3. 근본 원인 — 면접이 요구하는 설명 인출은 **결정적 판정**(허용답 문자열 대조)으로 담을 수 없다. 서술형 미도입의 이유가 그것이었다. 따라서 이 문제는 콘텐츠 수정이 아니라 "설명 인출을 어디에, 어떤 판정으로 담는가"의 기획 결정이다.
- **방향 — 두 가치의 계단 위계(DI1):** 핵심 가치 "CS 공부 진입장벽 낮추기"와 "면접 대비"를 경쟁시키지 않고 위계로 배치한다.
  - **1단(진입·용어 정착) = 기존 주관식 단답.** 5분·가볍게·결정적 판정. 진입장벽 낮추기는 여기가 계속 지킨다.
  - **2단(면접 가치) = 면접 세션.** 숙련도가 오른 개념부터 승급해, 면접 방향(이름→설명)의 인출을 AI 루브릭 채점으로 검증한다.
- **오너가 정의한 면접 가치:** "지금 배운 게 면접에서 유용하겠다는 믿음"(커버리지 확신). 빈도 통계가 아니라 **변환 시연**("이 문제가 면접에서는 이런 질문으로 나온다")과 **준비의 축적 가시화**(면접 준비도·잠금 해제)로 만든다.

## 3. 도메인 규칙 (확정 — 명세서 반영 대상)

### 3.1 A — 면접 연결 콘텐츠 (즉시 도입)

- **"면접에서는 이렇게 물어요" 블록.** 정답 처리 후 노출되는 심화 정보 안에, 그 문제의 개념이 실제 면접에서 어떤 질문 형태로 나오는지를 보여주는 항목을 둔다(예: items에 제목 고정 "면접에서는 이렇게 물어요" + 설명 "프로세스와 스레드의 차이를 설명해보세요 — 이때 ○○·△△를 짚으면 좋아요").
  - **1차는 큐레이션 관례로 구현한다(스키마 무변경).** 심화 정보의 기존 구조(title/body/items/quote)에 items 항목 하나로 얹는다. C단계에서 면접 질문 콘텐츠(§3.3)가 생기면 이 블록이 그 질문의 미리보기가 되므로, 그때 일급 필드 승격을 재검토한다.
  - 빈도 주장("자주 나와요")을 문구에 쓰지 않는다 — 변환 시연("이렇게 물어요")까지만 (DI6).
- **기출 코퍼스 = 내부 선정 기준.** `docs/dev/문제 데이터 생성 가이드.md`에 규칙을 추가한다: "신규 문제는 **기준 코퍼스 N종 중 M종 이상이 다루는 주제**에서 우선 역설계한다"(기본 3종 중 2종 — 코퍼스 목록은 오너가 Stage 0에서 확정: 공개 기술면접 대비 레포·시중 면접 대비 서적 목차 등). 사용자에게 라벨로 노출하지 않는다.
- **콘텐츠 재조정.** 신규 저작 배치는 유도형·비교형 비중을 상향한다(신규분 기준 목표 비율은 생성 가이드에 명기 — 운영 판단, 기존 문제 재저작은 비범위 §7).

### 3.2 D — 연결 문제 (다개념 하드 게이트)

- **정의 [개정 2026-07-22: DI12].** 연결 문제는 **큐레이터가 명시 속성으로 지정한 문제**다(`Problem.integration`, 기본 false) — 여러 개념을 실제로 잇도록 저작된 문제만 지정한다. ~~개념 2개 이상 태깅 = 연결 문제~~ 기준은 폐기했다: 다개념 태깅은 복습 축을 위한 것으로 풍부할수록 좋고, 게이트와 무관하다(DI12 근거는 §2 표 참고). 로드맵 '통합·시스템형 지식 문제'의 앞당김이며, **선행 간선(디딤 묶음)은 불필요**하다.
- **하드 게이트 [개정 2026-07-22: DI12].** 새 개념 문제 배정에서 **연결 문제(지정된 문제만)**는 **그 문제의 모든 개념에 대해 사용자의 학습 이력(개념 숙련도 행)이 존재할 때만** 후보가 된다. '학습했다'는 그 개념의 문제를 정답 처리(맞힘 또는 정답 공개 후 따라 입력)한 적이 있다는 뜻이다 — 승급 임계(레벨≥1)와 달리 게이트는 '만난 적'이면 충분하다. 미지정 문제는 개념 수와 무관하게 게이트 밖이다. 게이트로 새 개념 후보가 전부 빠지면 **D8 반복 폴백으로 내려간다**(막다른 길 없음 — 콘텐츠가 있는데 세션 생성이 실패하는 일은 없다).
- **계단 보장(구조 검증) [반영 2026-07-22: DI12].** 연결 문제로 지정하려면 ① 개념 2개 이상 태깅 ② **각 구성 개념이 단일 개념 문제로도 다뤄질 것**(계단 존재)을 결정적으로 검증한다 — 시드 로더(파일 단위, 위반 시 기동 실패)와 승인 게이트(파이프라인 반입 시)가 막는다. 계단 없는 연결 문제는 영구 잠금이 되므로 콘텐츠 결함으로 취급한다.
- **복습·폴백 무관.** 복습 편입은 이미 푼 문제의 재출제라 게이트를 항상 충족한다. D8 반복 폴백도 이미 푼 문제 대상이므로 무관. 게이트는 **새 개념 배정의 후보 필터에만** 작용한다.
- **기존 콘텐츠 영향.** 시드의 다개념 문제(현재 20개)가 게이트 대상이 되어 신규 사용자의 초기 배정 풀이 소폭 줄어든다 — 단일 개념 문제가 다수(87개)라 세션 구성에는 지장 없음. 회귀 테스트로 명시한다.
- **잠금 해제 연출.** 세션 진행 중 정답 처리로 어떤 연결 문제의 마지막 구성 개념이 채워지면, **세션 완료 화면**에 해제 안내를 담백하게 표시한다("○○·△△를 모두 배웠어요 — 이제 두 개념을 잇는 문제를 만날 수 있어요"). 완결 축하의 위계를 침범하지 않는다(디자인 04). 서버가 완료 응답에 해제 목록을 포함해 클라이언트가 재계산하지 않는다.

### 3.3 C — 면접 세션

- **도메인 용어(등록 대상).**
  - **면접 세션(Interview Session):** 승급된 개념에 대해 면접 방향(이름→설명)의 인출을 검증하는, 일일 학습 세션과 **별개의** 활동. 회원 전용·일일 제한.
  - **면접 질문(Interview Prompt):** 개념에 귀속되는 큐레이션 콘텐츠 — 질문 문구 + 모범 설명 + **루브릭**(꼭 짚어야 할 핵심 포인트 목록, 순서 보존). 승인 게이트 대상(초안/검수중/승인/반려 — 기존 승인 상태 모델 그대로). 1차는 개념당 1개를 기본으로 한다.
  - **면접 준비도(Interview Readiness):** 사용자·개념별 설명 검증 상태 — **미검증 / 부분(일부 포인트 충족) / 검증됨(전 포인트 충족)** + 최근 결과. 학습 상태(사용자 소유)에 귀속. 기존 숙련도와 독립된 축(DI4).
- **승급(후보 산정) — 결정적.** 면접 세션의 후보 개념 = **[개념 숙련도 레벨 ≥ 1] ∩ [승인된 면접 질문 존재]**. 출제 우선순위: 미검증 > 부분 > 검증됨(재도전 허용). 레벨이 0으로 떨어지면 후보에서 자연 이탈, 회복 시 복귀(별도 상태 관리 없음).
- **세션 구성.** 분량 소량 고정 — **기본 3문제**(상수 분리·운영 튜닝). 후보가 3개 미만이면 있는 만큼으로 구성(1개 이상이면 시작 가능). 후보 0이면 시작 불가 + 긍정 빈 상태 안내.
- **일일 제한(쿼터) — 비용 가드레일 조건부 규칙 발동.** 도메인으로 고정: **카운트 대상 = AI 채점 성공 호출이 포함된 면접 세션의 생성**, **차감 시점 = 세션 생성 시**, **리셋 경계 = KST 자정**(일반 세션의 하루 경계와 동일). 수치는 운영 — **기본 하루 1세션**. 채점 전량 실패로 끝난 세션은 차감하지 않는다(아래 폴백).
- **진행 흐름.** 질문 제시 → 사용자가 **자기 말로 설명을 작성**(텍스트) → 제출 → AI가 루브릭 대조 채점 → 결과 표시(포인트별 충족 체크리스트 + 한 줄 코멘트) → 모범 설명·전체 포인트 공개 → 다음 문제. 재제출은 없다(1문제 1채점 — 재도전은 다음 날 세션에서).
- **AI 채점 원칙.**
  - **루브릭 대조만.** AI는 "사용자의 설명이 각 핵심 포인트를 짚었는가"를 포인트별 충족 여부로만 판정한다. 자유 평가·지식 공급을 시키지 않는다 — 채점 기준은 전부 큐레이션(승인 게이트 통과) 콘텐츠다. P4(콘텐츠 신뢰) 노출 표면을 최소화하는 구조.
  - **무낙인 톤.** 점수·합불 선고가 아니라 "짚은 포인트 n개 / 보완하면 좋은 포인트"로 표현한다. 미충족은 비난이 아니라 다음 재도전의 안내다.
  - **폴백(graceful).** 채점 실패(타임아웃·오류, 재시도 1회 후) 시 모범 설명·핵심 포인트를 그대로 공개하고 준비도는 갱신하지 않는다. 세션의 모든 문제가 채점 실패면 쿼터를 차감하지 않는다.
  - **신고.** 채점 결과 화면에서도 콘텐츠 오류 신고를 허용한다(대상은 해당 면접 질문 콘텐츠).
- **결정적 판정 원칙과의 경계(명세 개정 핵심).** 코어 루프(주관식 문제 풀이·숙련도·복습)의 결정성은 **무손상 유지**된다. 비결정 판정(AI)은 **면접 세션 안에만** 존재하며, 그 산출물은 면접 준비도와 **복습 시점 당김(전용 예외 — §3.4 DI11)**에만 반영되고 **숙련도 레벨에는 반영되지 않는다**. "판정에 비결정성을 들이지 않는다"는 원칙은 "비결정성은 면접 채점 한 지점에 국한되며, **기록된 채점 결과로부터의 모든 파생은 결정적**"으로 경계를 재획정한다 — "같은 이력→같은 결과"의 '이력'에 면접 세션 결과 기록이 포함된다.
- **회원 전용.** 게스트는 진입 시 가입 안내를 받는다 — 승계 덕분에 문구가 구체적이다: "가입하면 지금까지 익힌 개념 N개를 면접처럼 연습할 수 있어요"(N = 게스트의 현재 후보 개념 수). 강요 없음(무낙인).
- **스트릭(DI5).** 면접 세션 완료도 스트릭 기록을 트리거한다. 하루 멱등 유지 — 같은 날 일반 세션과 합쳐 첫 완료 1회만 기록. 회귀 테스트 필수(면접 완료→기록 / 같은 날 일반 완료→중복 없음 / 역순 동일).
- **커버리지 표시(스트레치 — A-④ 이동분).** 카테고리별 "면접 질문이 준비된 개념 중 검증됨 n/m". 근거가 감당하는 주장("이 앱이 준비한 면접 주제 중")만 한다. C단계 후반 스트레치로, 1차 출시에 필수 아님.

### 3.4 주관식과의 동기 루프 (DI9~DI11 — 2026-07-22 확정)

> 승급 게이트(DI8)만으로는 구조적 연결일 뿐 사용자 체감 루프가 닫히지 않는다는 논의에서 확정. 루프: 주관식에서 스스로 맞힘 → [승급 가시화] → 면접 세션에서 설명 검증 → [미달 시 재열람 링크 + 이른 복습] → 주관식/복습 → ….

- **승급 순간 가시화(DI9).**
  - 주관식 **정답 처리 응답**에 그 제출로 새로 승급된 개념 목록을 포함하고(그 제출로 최초 레벨≥1 도달 ∧ 승인된 면접 질문 존재 — 결정적 계산), 정답 피드백 아래 **고정 인라인 라인**으로 표시한다: "이 개념의 면접 연습이 열렸어요". 토스트·스낵바는 미채택(사라지는 정보·낭독 처리·신규 인프라 비용).
  - **세션 완료 응답**에 그 세션에서 승급된 개념 집합 `newlyEligibleConcepts`를 포함하고, 완료 화면에 요약 카드로 표시한다(완결 축하 아래, D안 잠금 해제 배지와 동일 위계·담백한 톤).
  - 게스트에게도 동일하게 계산·표시한다 — 면접 세션 진입 시 가입 안내로 이어지는 자연 훅("익힌 개념 N개…"와 정합).
- **역방향 연결(DI10).** 면접 세션 결과 화면에서 '검증됨' 미달(부분·미충족) 개념에 대해 ① **'그때 푼 문제 다시 보기'** — 그 개념으로 푼 문제의 읽기 전용 재열람(스크랩·카테고리 이력과 동일 패턴 재사용) ② **"복습에서도 곧 다시 만나요"** 안내를 병기한다. 강제·자동 이동 없음(무낙인).
- **복습 당김 전용 예외(DI11 — DI4 개정).** 면접 세션에서 '검증됨' 미달로 기록된 개념은 해당 개념 숙련도의 다음 복습 시점을 `min(기존 예정일, 면접일+1일)`로 갱신한다.
  - **당김만 있다** — 기존 예정일이 더 이르면 무변경, 미루는 방향의 갱신은 없다.
  - **숙련도 레벨·간격 사다리 위치는 무변경** — 당겨진 복습에서의 정답은 기존 규칙(D8 포함)대로 처리된다.
  - **채점 실패(폴백) 시 무변경** — 준비도 미갱신과 동일 원칙.
  - 결정성: 당김은 기록된 면접 결과로부터의 **결정적 파생**이다(위 '경계' 문단). AI 오채점의 피해 상한은 '잘 아는 개념을 하루 일찍 복습'으로, 학습에 무해한 방향으로만 흐른다.
- **2단 사다리 표시(용어 정착→설명 검증 칩)는 1차 화면에 넣지 않는다** — C5(커버리지 표시)로 전량 이연.

## 4. 시스템 변경 설계

### 4.1 DB — Flyway (⚠️ V7=난이도 1차(적용됨), **V8=연결 문제 속성(DI12, D1 재작업에서 사용)** — 아래 면접 세션 테이블은 **V9부터**)

```sql
-- 면접 질문(개념 귀속 콘텐츠) + 루브릭
CREATE TABLE interview_prompt (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  concept_id BIGINT NOT NULL REFERENCES concept(id),
  question TEXT NOT NULL,
  model_answer TEXT NOT NULL,
  approval_status VARCHAR(16) NOT NULL
);
CREATE TABLE interview_prompt_rubric_point ( -- @ElementCollection, position 순서 보존
  prompt_id BIGINT NOT NULL REFERENCES interview_prompt(id),
  position INT NOT NULL,
  text TEXT NOT NULL
);
-- 면접 세션·항목
CREATE TABLE interview_session (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id BIGINT NOT NULL REFERENCES users(id),
  session_date DATE NOT NULL,
  status VARCHAR(16) NOT NULL,
  started_at timestamp(6) with time zone,
  completed_at timestamp(6) with time zone
);
CREATE INDEX idx_interview_session_user_date ON interview_session(user_id, session_date);
CREATE TABLE interview_session_item (
  session_id BIGINT NOT NULL REFERENCES interview_session(id),
  position INT NOT NULL,
  prompt_id BIGINT NOT NULL REFERENCES interview_prompt(id),
  submitted_text TEXT,
  satisfied_points TEXT,      -- 충족 포인트 position 목록(직렬화) 또는 별도 테이블 — 구현 판단
  judge_comment TEXT,
  judged_at timestamp(6) with time zone
);
-- 면접 준비도(학습 상태)
CREATE TABLE interview_readiness (
  user_id BIGINT NOT NULL REFERENCES users(id),
  concept_id BIGINT NOT NULL REFERENCES concept(id),
  status VARCHAR(16) NOT NULL, -- UNVERIFIED/PARTIAL/VERIFIED
  satisfied_count INT NOT NULL,
  total_count INT NOT NULL,
  updated_at timestamp(6) with time zone NOT NULL,
  CONSTRAINT uk_interview_readiness UNIQUE (user_id, concept_id)
);
```
- 쿼터는 별도 테이블 없이 `interview_session`의 (user_id, session_date) 카운트로 도출한다(기본 하루 1이면 존재 여부 검사).

### 4.2 서버

- **새 슬라이스** `interview.{domain,application,presentation,infrastructure}` — Spring 아키텍처 가이드 준수. 계정 삭제 시 면접 세션·준비도 삭제 편입(학습 상태), 면접 질문 콘텐츠는 공용 자산이라 무관.
- **AI 채점 클라이언트:** 도메인 인터페이스 `ExplanationJudge`(입력: 루브릭 포인트 목록+사용자 설명 / 출력: 포인트별 충족 boolean+한 줄 코멘트, 구조화 응답 강제). 구현체는 LLM API 호출(모델·키는 환경변수 — 공급자·모델 선택은 구현 시 결정). **local·test 프로파일은 결정적 Fake 구현**(예: 포인트 문자열 포함 여부 매칭) — `:server:test`에서 실호출 0.
- **API:**
  - `GET /api/interview/status` — 후보 개념 수·오늘 잔여 쿼터·게스트 여부(홈 카드 단일 출처. 게스트에게는 후보 수만 계산해 가입 유도 문구용으로 제공).
  - `POST /api/interview/sessions/today` — 생성(후보 없음 404류·쿼터 소진 409류·게스트 403, 에러코드 구분).
  - `GET /api/interview/sessions/today` — 진행 중 세션 조회(중단·재개 — 일반 세션과 동일하게 영속).
  - `POST /api/interview/sessions/today/answers` — 제출·채점(폴백 시 응답에 judged=false + 모범 설명·포인트 포함).
  - 완료 응답에 스트릭 포함(recordStudy — 하루 멱등은 기존 로직 재사용, lockUser 직렬화 관례 준수).
- **D 게이트:** SessionCreator의 새 개념 후보 필터에 "concepts.size ≥ 2 → 전 개념 mastery 행 존재" 조건 추가 + 세션 완료 감지 지점에서 신규 해제 연결 문제 계산 → 완료 응답 `unlockedIntegrations: [{problemSummary, concepts}]`.
- **동기 루프(DI9~DI11):** 주관식 정답 처리 응답에 `newlyEligibleConcepts`(그 제출로 최초 레벨≥1 도달 ∧ 승인 면접 질문 존재 — MasterySignal 반영 직후 결정적 계산) + 세션 완료 응답에 세션 단위 집계. 면접 채점 확정 시 '검증됨' 미달 개념의 `ConceptMastery.nextReviewDate = min(기존, 면접일+1)` 갱신(레벨·사다리 무변경 — 전용 메서드로 의도 고정, 벌크 아닌 애그리거트 경유).
- **보안:** `/api/interview/**`는 authenticated + **MEMBER 역할**(게스트 403 + 전용 에러코드). `/status`만 게스트 허용(가입 유도 데이터).

### 4.3 클라이언트 (Compose Multiplatform — android/wasmJs 공통)

- **홈(디자인 02):** 면접 세션 진입 카드 — 상태 4종: (게스트) 가입 유도 문구 / (후보 0) 긍정 빈 상태 "오늘의 한입에서 개념을 스스로 맞히면 면접 연습이 열려요" / (잔여 있음) 진입 CTA "익힌 개념 N개, 면접처럼 설명해보기" / (오늘 소진) "내일 다시 열려요" 담백 안내. 오늘의 한입 카드보다 아래 위계(1단이 히어로).
- **면접 세션 화면(신규 — 디자인 08):** 질문 → 멀티라인 입력(주관식과 다른 촉감 — "내 말로 설명해보세요" 플레이스홀더) → 제출 → 결과(포인트 체크리스트: 짚은 것 ✓·보완할 것 안내 톤) → 모범 설명 → 다음. 채점 중 로딩 상태(수 초) 명시적 처리. 폴백 시 "채점을 잠시 쉬어갈게요 — 모범 설명과 비교해보세요". '검증됨' 미달 개념 결과에는 **'그때 푼 문제 다시 보기' 링크 + "복습에서도 곧 다시 만나요" 병기**(DI10).
- **문제 풀이 화면(디자인 03):** 정답 피드백 아래 승급 **고정 인라인 라인** "이 개념의 면접 연습이 열렸어요"(DI9 — 응답의 newlyEligibleConcepts 존재 시에만, 담백한 톤·라이브리전 낭독 1회).
- **세션 완료 화면(디자인 04):** `unlockedIntegrations` 있으면 잠금 해제 배지 + `newlyEligibleConcepts` 있으면 면접 연습 열림 요약 카드(둘 다 완결 축하 아래, 담백 — DI9).
- **심화 "면접에서는 이렇게 물어요"**: 콘텐츠 관례라 클라 무변경(기존 items 렌더 재사용).

### 4.4 AI 채점 설계 세부

- 프롬프트: 시스템 지시(루브릭 대조 채점자 역할·JSON 스키마 고정·포인트 외 평가 금지) + 루브릭 포인트 + 사용자 설명. 낮은 온도. 응답 파싱 실패는 채점 실패로 취급(폴백).
- 비용 상한 = 쿼터가 캡: 사용자당 하루 1세션 × 3문제 = 채점 3콜. 모델 선택·단가 검토는 Stage C3에서(환경변수로 교체 가능하게).
- 채점 결과 로깅(포인트 충족 분포·실패율)은 /admin/stats 확장으로 — 2차 튜닝(검증됨 기준·쿼터 수치) 근거.

## 5. 구현 단계와 커밋 계획

각 커밋은 서비스 온전 동작 + 테스트 통과 전제(버전 관리 지침). A → D → C 순서로 가치가 빨리 나오는 것부터.

| Stage | 내용 | 커밋(안) |
|-------|------|---------|
| 0 | 명세서·디자인·생성 가이드 갱신(§6) + 본 계획 문서 | `docs: 면접 대비 가치(면접 연결 콘텐츠·연결 문제·면접 세션) 명세 반영` |
| A1 | 생성 가이드에 기출 코퍼스 규칙·심화 블록 관례·유형 비율 명기(코퍼스 목록은 오너 확정 반영) | `docs: 문제 생성 가이드에 면접 역설계 기준 추가` |
| A2 | 기존 107문제 심화에 "면접에서는 이렇게 물어요" 블록 백필(별도 저작 세션 가능 — 코퍼스 기준 적용) | `feat: 심화 정보에 면접 질문 변환 블록 백필` |
| D1 | 연결 문제 게이트 **[개정: DI12]** — V8(integration 컬럼)·시드 스키마(선택 필드·하위 호환)·구조 검증 2건(개념 2+ 필수·계단 보장)·플래그 기반 SessionCreator 필터·게이트 공동 시 D8 폴백 + 회귀 테스트 | `feat: 연결 문제 출제 게이트(구성 개념 학습 후 출제)` |
| D1-콘텐츠 | v2 시드 `integration` 백필(저작 의도 기준 판별) + 계단 보강 단일 문제 소량 저작(플래그 문제의 미커버 구성 개념) — 기존 107문제 시드는 백필 보류(플래그 없음=게이트 밖, 병행 테스트 무영향) | `feat: v2 시드 연결 문제 지정·계단 보강` |
| D2 | 잠금 해제 계산 + 완료 응답 + 완료 화면 배지 | `feat: 연결 문제 잠금 해제 안내` |
| C1 | 면접 질문 도메인·마이그레이션·시드 반입·승인 게이트 편입(구조 검증: 루브릭 1개 이상·모범 설명 존재) | `feat: 면접 질문 콘텐츠 도메인` |
| C2 | 면접 세션 도메인+API(Fake 채점기)+승급 산정+쿼터+준비도+스트릭 OR+계정 삭제 편입+**승급 가시화 응답(DI9)·복습 당김(DI11)** | `feat: 면접 세션 도메인·API` |
| C3 | 실 AI 채점 클라이언트(환경변수·재시도·폴백) — local/test는 Fake 유지 | `feat: 면접 설명 AI 루브릭 채점 연동` |
| C4 | 클라 홈 카드 + 면접 세션 화면 + 완료 화면 + **승급 인라인 라인·요약 카드(DI9)·재열람 링크(DI10)** | `feat: 면접 세션 화면` |
| C5 | (스트레치) 카테고리별 면접 커버리지 표시 + /admin/stats 채점 지표 | `feat: 면접 준비 커버리지·지표` |

### 테스트 관점 (필수 커버)
- **D 게이트:** 다개념 문제가 미학습 개념 포함 시 후보 제외 / 전 개념 학습 시 포함 / 단일 개념 무영향 / 복습·반복 폴백 무관 / 해제 목록 계산(그 세션에서 마지막 개념이 채워진 경우만).
- **승급 산정:** 레벨 0(도움만)·미학습 제외, 레벨 1 포함, 레벨 하락 시 이탈 / 승인 면접 질문 없는 개념 제외 / 우선순위(미검증>부분>검증됨).
- **쿼터:** 생성 차감·KST 자정 리셋·소진 409·전량 채점 실패 시 미차감 / 게스트 403·/status는 게스트 후보 수 제공.
- **준비도:** 포인트 충족 수→상태 파생(부분/검증됨) / **기존 ConceptMastery·복습 스케줄 무변경**(핵심 회귀 — DI4) / 채점 실패 시 미갱신.
- **스트릭 OR:** 면접 완료→기록 / 같은 날 일반 완료→중복 없음 / 역순 동일 / 하루 멱등.
- **Fake 채점기:** 결정적 동작 검증 + 실 구현은 계약 테스트(구조화 응답 파싱·실패 폴백)만.
- **동기 루프:** DI9 승급 계산(그 제출로 **최초** 레벨≥1 도달 ∧ 승인 면접 질문 존재일 때만 표시·이미 승급된 개념 재정답 시 미표시·완료 응답은 세션 단위 집계), DI11 당김 전용(기존 예정일이 더 이르면 무변경·**미루는 방향 절대 없음**·숙련도 레벨/사다리 무변경·채점 실패 시 무변경 — 핵심 회귀).

## 6. 명세서·문서 갱신 목록 (Stage 0)

- **도메인 명세서**
  - 도메인 용어: **면접 세션 · 면접 질문 · 루브릭(핵심 포인트) · 면접 준비도 · 연결 문제** 등록.
  - 핵심 목적·페인포인트: 수단 목적에 면접 가치 계단 반영, P6 서술에 "면접 유용성 믿음" 연결(또는 페인포인트 신설 — 기획 판단).
  - 기능 1: 결정 근거의 "정의 재생형에 성립하지 않는다" 서술에 "면접 세션이 설명 방향을 전담해 회수한다 [반영 2026-07-21: DI1]" 갱신. 서술형 미도입 결정에 "설명 인출은 면접 세션(AI 루브릭 채점)으로 별도 수용 — 코어 루프 단답 유지" 명기.
  - 기능 1.5 문제 선정: 연결 문제 하드 게이트(§3.2) 추가.
  - 신규 기능 절(예: 기능 8 면접 세션): §3.3 전체 — 목적·가치·승급·쿼터·채점 원칙·준비도·수용 기준.
  - 결정적 판정 원칙(기능 1·콘텐츠 가드레일): 경계 재획정 — "학습 상태의 결정적 축(숙련도·복습)에 비결정성을 들이지 않는다. AI 채점은 면접 세션 한정, 산출물은 면접 준비도에만".
  - 비용 가드레일: 조건부 미래 규칙 **발동** — 카운트 대상·차감 시점·리셋 경계(§3.3) 도메인 고정, 수치는 운영.
  - 기능 3(복습): 복습 시점의 **당김 전용 예외(DI11)** 명기 — 면접 세션 결과 기록에서 결정적으로 파생, 숙련도 레벨·간격 사다리 무변경, 미룸 없음.
  - 기능 6: 스트릭 OR 조건(DI5).
  - 기능 8(면접 세션): 동기 루프(DI9 승급 가시화·DI10 역방향 연결)를 수용 기준에 포함.
  - MVP 경계·로드맵: '통합·시스템형' → D로 [구현] 전환, 'AI 의미 채점' → C로 [구현] 전환(음성·모의 면접 리허설은 로드맵 잔존).
  - 열린 질문: 쿼터 수치·세션 분량 튜닝 / 검증됨 기준(전 포인트 vs 임계) / 기준 코퍼스 목록·M/N / 면접 질문의 개념당 복수화 / 재도전 정책(당일 재시도 허용 여부).
- **디자인 문서:** 02 홈 카드(상태 4종), 03 정답 피드백 승급 인라인 라인(DI9), 04 완료 잠금 해제 배지+면접 열림 요약 카드(DI9), **08 면접 세션 화면 신설**(결과의 재열람 링크 포함 — DI10), DESIGN_SYSTEM 신규 컴포넌트(체크리스트 결과·채점 로딩).
- **문제 데이터 생성 가이드:** 기출 코퍼스 규칙·심화 블록 관례·신규 유형 비율·연결 문제 저작 기준(개념 2개 이상 태깅 시 게이트가 걸림을 저작자가 인지)·면접 질문(루브릭) 저작 스키마.

## 7. 비범위 (이번 계획에서 하지 않는 것)

- 문제별 "면접 빈출" 배지 등 **빈도 주장 일체**(DI6 폐기 — 재검토 조건: 검증 가능한 실측 데이터 확보 시).
- 셀프 체크(자기 채점) — DI2 기각.
- 음성 답변·모의 면접 리허설(대화형) — 로드맵.
- 기존 87개 단일 개념 문제의 유형 재저작(신규 저작 기준만 변경).
- 게스트 면접 세션(회원 전용 확정 — 맛보기 도입은 어뷰징 방어 설계와 함께 재검토).
- 면접 준비도의 간격 반복(자동 재출제 스케줄) — 1차는 사용자 주도 재도전만.
- AI 실시간 힌트 등 다른 런타임 AI 확장(이 계획의 쿼터 설계를 선례로 그때 별도 결정).
