# 웹(wasmJs) 테스트 가이드

웹 클라이언트(Compose Multiplatform wasmJs)는 캔버스 렌더링·브라우저 IME·같은 오리진 서빙 등
**헤드리스 자동화로 잡히지 않는 표면**이 있다. 이 문서는 웹 관련 변경 후 실제 브라우저에서 돌리는
스모크 체크리스트다. 자동 테스트(`:app:shared:jvmTest`의 UI 테스트, `:server:test`)를 대체하지 않고 보완한다.

> 원칙: 코드 변경은 늘 `:server:test` + `:app:shared:jvmTest` + `:app:shared:compileKotlinWasmJs`가 green이어야
> 하고(공통 게이트), 웹 표면을 건드렸다면 아래 브라우저 스모크를 **추가로** 돌린다.

## 1. 실행 방법

두 가지 경로 중 하나로 띄운다.

- **개발(핫 리로드)**: `gradlew :app:webApp:wasmJsBrowserDevelopmentRun` → `http://localhost:8081`
  - 로컬 Spring이 8080에 떠 있어야 한다(`gradlew :server:bootRun`, local 프로파일). `/api`는 프록시로 전달된다.
- **통합(배포와 동일한 같은 오리진 서빙)**: `gradlew -PincludeWeb :server:bootRun` → `http://localhost:8080`
  - 웹 프로덕션 번들이 서버 `static/`으로 들어가 프록시 없이 한 오리진에서 서빙된다. 배포에 가장 가깝다.
  - 프로덕션 번들 빌드(Binaryen 최적화)는 수 분 걸린다.

## 2. ⭐️ 최우선: 한글 IME 조합 입력

주관식 한글 답변이 서비스의 핵심 입력이다. **여기서 결함이 발견되면 진행을 멈추고 오너에게 보고**한다.

- [ ] 문제 풀이 답변 `TextField`에 한글을 조합 입력한다(예: "해시 충돌").
- [ ] 조합 중 백스페이스(받침 지우기 → 초성만 남기기)가 자연스럽다.
- [ ] Chrome에서 확인하고, 가능하면 Safari에서도 확인한다(엔진별 IME 차이).

## 3. 핵심 플로우 스모크

- [ ] 첫 진입: 게스트 자동 발급 → 온보딩 → 홈('오늘의 한입')이 뜬다.
- [ ] 오늘의 세션 시작 → 정답/오답 제출 → 피드백 표시.
- [ ] 힌트 공개, 정답 공개 후 따라 입력.
- [ ] 스크랩 추가/해제 → 스크랩 목록에 반영.
- [ ] 카테고리 이력 목록 → 상세 → 개별 문제 열람.
- [ ] **새로고침 후 토큰·학습 상태 유지**(localStorage). 로그인 없이도 같은 게스트로 이어진다.
- [ ] DevTools Network에서 `/api` 요청에 `Authorization: Bearer` 헤더가 붙는다.
- [ ] 마지막 문제 정답 후 '한입 마치기' → 완료 화면.

## 4. 웹 입력·상호작용

- [ ] 정답 후 하단 버튼이 '다음 문제'로 바뀌면 **Enter로** 다음 문제로 넘어간다(클릭 불필요).
- [ ] 이때 이중 발화(한 번의 Enter로 두 문제 건너뜀)가 없다.
- [ ] 답변 입력 중 Enter로 제출된다(제출 가능 상태일 때).
- [ ] Tab으로 버튼 포커스 이동·포커스 표시, 마우스 휠 스크롤, 텍스트 선택이 동작한다.

## 5. 적응형 레이아웃(창 너비)

브라우저 창 폭을 바꿔가며 확인한다(개발자도구 반응형 모드 또는 창 크기 조절).

- [ ] **좁게(<600dp, COMPACT)**: 모바일과 동일한 세로 단일 컬럼.
- [ ] **넓게(≥840dp, EXPANDED)**:
  - [ ] 홈이 2컬럼(좌 오늘의 한입 카드 / 우 스트릭·진입점)으로 재배치된다.
  - [ ] 스크랩·카테고리 이력이 좌 목록·우 상세 2패널로 뜨고, 목록 선택 시 우측 패널만 바뀐다(뒤로가기 없이).
  - [ ] 문제 풀이는 재배치 없이 본문·입력·CTA가 가독폭(≈720dp)으로 중앙 유지된다(화면 끝까지 늘어지지 않음).
  - [ ] 계정/로그인/온보딩/세션완료는 중앙 카드로 정돈된다.

## 6. 로딩·미지원 브라우저

- [ ] 첫 로드 시 서비스명 + 스피너 로딩 화면이 뜨고, 앱이 준비되면 사라진다.
- [ ] 지원 브라우저(WasmGC): Chrome 119+ / Firefox 120+ / Safari 18.2+ 에서 정상 부팅.
- [ ] 미지원(구형) 브라우저: "최신 Chrome/Firefox/Safari에서 열어 주세요" 안내가 뜨고 20MB 번들을 내려받지 않는다.
- [ ] 파비콘·탭 제목(CS한입)이 표시된다.
- [ ] 한글·아이콘이 정상 렌더(Pretendard 폰트 번들 + vector 아이콘 — 캔버스에서 이모지/시스템 폰트는 안 됨).

## 7. 알려진 제약(백로그)

- 브라우저 뒤로가기(popstate)는 앱 백스택과 무관하다(탭 이탈). URL 라우팅/딥링크 없음.
- 웹 접근성(스크린리더)은 CMP 웹의 알려진 제약으로 실효성이 제한적이다.
- 첫 로드 번들이 크다(폰트 subsetting·캐싱 헤더는 배포 최적화 백로그).
