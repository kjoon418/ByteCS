# 실기기 QA 개선 — 작업 개요 (2026-07-17)

실기기 QA에서 나온 개선사항 8건을 맥락별로 4개 작업 단위로 묶고, 우선순위·개발 비용을 고려해 순서를 정했다.
각 작업 단위의 구체 지시는 같은 디렉터리의 `01`~`04` 문서를 따른다.

| 3 | [03 홈 CTA 버튼 카드 내부 이동](./03%20홈%20CTA%20버튼%20이동.md) | #8 | UI 개선 | 소 |

## 순서 근거
- **1순위 (이메일 오류):** 검증 실패를 "연결 실패"로 오안내해 신뢰를 깎는 명백한 버그이며, 수정 범위가 좁다(클라 3지점 + 서버 테스트 보강). 가장 싸게 가장 확실한 결함을 제거한다.
- **2순위 (문제 선정):** "추가 연습에서 같은 문제 하나만 반복"은 기능이 사실상 동작하지 않는 버그이고, 일일 세션 랜덤화는 학습 경험의 핵심 요구다. 두 항목 모두 서버 문제 선정 로직이라는 같은 맥락이라 한 단위로 묶었다. 도메인 명세 개정(§1.5 추가 연습 규정 신설 등)이 동반된다.
- **3순위 (홈 CTA):** 최신 디자인 시안(`docs/design/02 홈 오늘의 한입 디자인 최신안.html`)의 버튼 배치를 따르는 국소 UI 변경으로, 비용이 작아 큰 작업(4순위) 전에 빠르게 끝낸다.
- **4순위 (세션 화면 UX):** 4개 QA 항목이 모두 세션 화면(`SessionScreen.kt`)에 얽혀 있어 한 단위로 묶되, 내부적으로 6개 커밋으로 쪼갠다. 서버 정답 공개 정책 완화와 도메인 명세 개정을 포함해 가장 크고, 앞 작업들과 독립적이므로 마지막에 진행한다.

## 사용자 확정 사항 (2026-07-17 문답)
- **오답 이펙트 색:** 무낙인 원칙 유지, **주황 계열** 톤으로 카드 플래시.
- **정답 보기:** **시도(오답 제출) 전에도 허용** — 서버 가드·도메인 명세 함께 개정.
- **홈 CTA:** `02 홈 오늘의 한입 디자인 최신안.html`의 '학습 이어서 하기' 버튼(카드 내부 배치) 부분만 참고. 최신안의 나머지 요소(하단 탭 바, '오늘의 세션' 배지, 미래형 스트릭 문구)는 이후 오너 결정으로 폐기된 것이므로 반영하지 않는다.

## 공통 규칙
- 각 작업 단위는 CLAUDE.md 버전 관리 규칙을 따른다: 서비스가 온전히 동작하는 상태로 커밋을 나누고, 프로덕션 코드에는 테스트를 동반하며, AngularJS 커밋 컨벤션을 쓴다.
- UI 문자열·색상은 기존 규칙 유지: raw hex 금지(`LocalBcsColors` 토큰), 무낙인 원칙(빨강·'오답/틀림/실패' 문구 금지), 내부 용어 '세션' 미노출.
- 검증: 서버 `./gradlew test`(server), 클라 `./gradlew jvmTest`(app) 전체 통과 + 해당 화면 수동 확인.
