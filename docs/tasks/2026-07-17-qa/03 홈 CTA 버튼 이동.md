# 03. 홈 CTA 버튼을 현황 카드 안으로 이동 (QA #8)

## 배경
홈 화면에 '시작하기' 성격의 요소가 두 곳(현황 카드의 상태 텍스트 + 하단 고정 바 버튼)에 나뉘어 있다. CTA 버튼을 '오늘의 한입' 현황 카드 안으로 옮겨, 버튼을 누르면 무엇이 시작되는지 직관적으로 알 수 있게 한다.

## 기준 시안 (2026-07-17 사용자 확정)
`docs/design/02 홈 오늘의 한입 디자인 최신안.html`의 **'학습 이어서 하기' 버튼 부분만** 참고한다.
- 버튼은 현황 카드(`#main-session`) 내부, **진행 막대 아래**에 전폭(full-width) 배치 (최신안 62~66행). 버튼 아래에 복습 힌트 문구가 온다.
- ⚠️ 최신안의 나머지 요소는 반영 금지 — 하단 탭 바(폐기), '오늘의 세션' 배지(내부 용어 '세션' UI 노출 금지 규칙 위반), 미래형 스트릭 문구("내일도 오시면…", 손실 프레임으로 폐기됨), 프로필 사진, "2 / 5" 분량. 이들은 기존 시안(`02 홈 오늘의 한입 디자인.html`)과 오너 결정이 우선한다.

## 현재 구조 (확인 완료)
`app/shared/src/commonMain/kotlin/watson/bytecs/session/HomeScreen.kt`
- 현황 카드: `TodayBiteCard`(288~342행) — 배지·제목·진행 카운트·`ProgressBar`(379~403행)·복습 힌트.
- CTA: `BcsScaffold`의 `bottomBar` 슬롯(117~128행)에서 상태 분기 → `HomeCtaBar`(206~220행)가 감쌈.
  - 시작 전: `PrimaryButton("오늘의 한입 시작하기")` / 진행 중: `PrimaryButton("학습 이어서 하기")` / 완료: `GhostButton("조금 더 풀어보기")`

## 변경 사항
1. `TodayBiteCard` 내부, `ProgressBar` 아래(복습 힌트 문구 위)에 CTA 버튼을 배치한다. 기존 상태 분기(버튼 텍스트·종류 3종)와 콜백을 그대로 카드 안으로 옮긴다.
2. `HomeCtaBar`와 `bottomBar` 슬롯 사용을 제거한다. 하단 여백/스크롤 패딩이 어색해지지 않는지 확인.
3. 색상은 raw hex 금지 규칙에 따라 기존 `PrimaryButton`/`GhostButton` 컴포넌트와 `LocalBcsColors` 토큰을 그대로 쓴다(최신안의 `#1E293B` 네이비를 직접 쓰지 않는다 — 컴포넌트 기본 스타일 유지).
4. `HomeScreen.kt`(및 `HomeScreenUiTest.kt` 14행)의 시안 참조 주석에 "CTA 배치는 최신안(`02 홈 오늘의 한입 디자인 최신안.html`)을 따른다"를 덧붙인다.

## 테스트 (`HomeScreenUiTest.kt`)
- 기존 버튼 텍스트 단언("오늘의 한입 시작하기" 96행, "학습 이어서 하기" 85행, "조금 더 풀어보기" 120행)은 위치만 바뀌므로 그대로 통과해야 한다.
- 추가: CTA 버튼이 현황 카드 영역 안에 있음을 좌표(`getBoundsInRoot`)로 검증하는 케이스 1건 — 세션 화면 테스트의 `모범답안은_따라_입력_칸보다_위에_있다` 패턴 참고.
