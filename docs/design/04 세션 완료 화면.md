**전제:** `00 공통 디자인 원칙.md`. 이 문서는 이 페이지의 목적·구성 요소만 기술한다.

# 1. 어떤 페이지인가
- **세션 완료 화면** — 오늘의 한입(배정된 본 문제)을 다 마쳤을 때 뜨는 완결 화면.
- 도메인 근거(기능 1.5): 세션은 본 문제 분량을 마치면 **완료로 전이**하고 **완결 신호**("오늘 CS 한입 완료")를 준다.
- 목적: **작은 성취의 완결감**을 주어 지속(내일 또) 동기를 만든다.

# 2. 이 페이지의 테마 강조점
- **따뜻한 성취·격려.** "오늘도 한입 해냈어요!" 과하지 않게, 기분 좋게.
- 정착의 연결감(오늘 배운 것이 나중에 복습으로 다시 온다)을 은은하게.
- 부담·의무감 없이 마무리. 다음 방문을 **강요 아닌 초대**로.

# 3. 구성 요소 (구체)
1. **완결 축하 헤더 + 메타포** — "오늘 CS 한입 완료!" + 완결감 그래픽/짧은 성공 애니메이션(시각 + 햅틱).
2. **오늘의 요약(가볍게)** — 오늘 푼 본 문제 수, (선택) 새로 만난 개념 수 / 복습한 개념 수. 숫자 압박 아닌 성취 표현.
3. **정착 연결 안내(은은)** — "오늘 배운 건 나중에 복습으로 다시 만나요." (간격 반복 암시, 부담 X)
4. **연속 학습(스트릭) — 선택·긍정** — 이어졌으면 격려. 끊김 죄책감 연출 금지.
5. **Primary CTA — [오늘은 여기까지]** — 홈으로. 편안한 마무리.
6. **Secondary — [조금 더 풀기]** — 추가 학습(압박 아닌 선택).
7. **게스트 승계 유도(게스트일 때만·은은)** — "가입하면 이 기록이 사라지지 않아요." → 05로. 성취의 순간에 자연스러운 권유(강요 X).

# 인터랙션 흐름
1. 03에서 마지막 본 문제 완료 → 이 화면.
2. **[오늘은 여기까지]** → 02 홈(완료 상태). / **[조금 더 풀기]** → 03.

# 이 페이지 주의
- ⛔ "내일 안 하면…" 식 죄책감·불안 유발 금지.
- 게스트 승계는 축하 맥락의 **자연스러운 권유**로만.
