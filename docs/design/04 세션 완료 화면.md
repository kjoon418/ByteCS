**전제:** `00 공통 디자인 원칙.md`. 이 문서는 이 페이지의 목적·구성 요소만 기술한다.

# 1. 어떤 페이지인가
- **세션 완료 화면** — 오늘의 한입(배정된 본 문제)을 다 마쳤을 때 뜨는 완결 화면.
- 도메인 근거(기능 1.5): 세션은 본 문제 분량을 마치면 **완료로 전이**하고 **완결 신호**("오늘 CS 한입 완료")를 준다.
- 목적: **작은 성취의 완결감**을 충분히 주어 지속(내일 또) 동기를 만든다. 타깃(공부 의지 낮음, P5)에게 재접속의 핵심 동력이므로 **성취 연출을 아끼지 않는다**(단 긍정 톤, 기능 6).

# 2. 이 페이지의 테마 강조점
- **확실한 성취·격려(리뷰 반영).** "오늘도 한입 해냈어요!" — 성취감이 **충분히 느껴지도록** 축하 애니메이션·효과·햅틱을 넉넉히 준다(밋밋함이 문제였음). 단 과한 소음·자기목적적 연출이 아니라 '해냈다'는 보람에 복무.
- 정착의 연결감(오늘 배운 것이 나중에 복습으로 다시 온다)을 은은하게.
- 부담·의무감 없이 마무리. 다음 방문을 **강요 아닌 초대**로(죄책감·상실 공포 금지).

# 3. 구성 요소 (구체)
1. **완결 축하 헤더 + 메타포 + 축하 연출(리뷰 반영)** — "오늘 CS 한입 완료!" + 완결감 그래픽 + **눈에 확 들어오는 축하 애니메이션**(예: 컨페티·체크 모션·수치 카운트업) + 햅틱. 성취가 분명히 느껴지게(단 1~2초, 다음 행동을 막지 않게).
2. **오늘의 요약(가볍게)** — 오늘 푼 본 문제 수, (선택) 새로 만난 개념 수 / 복습한 개념 수. 숫자 압박 아닌 성취 표현.
3. **정착 연결 안내(은은)** — "오늘 배운 건 나중에 복습으로 다시 만나요." (간격 반복 암시, 부담 X)
4. **연속 학습(스트릭) — 긍정 동기(기능 6)** — 스트릭 상승을 성취로 또렷이 보여 재접속 동기를 만든다. ⚠️ **끊김에 죄책감·상실 공포 연출 금지**(끊겨도 "다시 시작해요" 톤). 강제 아님.
5. **Primary CTA — [오늘은 여기까지]** — 홈으로. 편안한 마무리.
6. **Secondary — [조금 더 풀기]** — 추가 학습(압박 아닌 선택).
7. **게스트 승계 유도 — 홈으로 일원화(2026-07-16 오너 결정)** — 이 화면에는 두지 않는다. 완료의 순간은 성취에만
   집중하고, 가입 접점은 홈의 가입 유도(GuestUpgradeBanner) 하나로 모은다(접점 분산 시 어디서도 강하게
   전달되지 않는 문제 방지).
