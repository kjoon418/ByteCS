**전제:** `00 공통 디자인 원칙.md`. 이 문서는 이 페이지의 목적·구성 요소만 기술한다.

# 1. 어떤 페이지인가
- **세션 완료 화면** — 오늘의 한입(배정된 본 문제)을 다 마쳤을 때 뜨는 완결 화면.
- 도메인 근거(기능 1.5): 세션은 본 문제 분량을 마치면 **완료로 전이**하고 **완결 신호**("오늘 CS 한입 완료")를 준다.
- 목적: **작은 성취의 완결감**을 충분히 주어 지속(내일 또) 동기를 만든다. 타깃(공부 의지 낮음, P5)에게 재접속의 핵심 동력이므로 **성취 연출을 아끼지 않는다**(단 긍정 톤, 기능 6).

# 2. 이 페이지의 테마 강조점
- **확실한 성취·격려(리뷰 반영).** "오늘도 한입 해냈어요!" — 성취감이 **충분히 느껴지도록** 축하 애니메이션·효과·햅틱을 넉넉히 준다(밋밋함이 문제였음). 단 과한 소음·자기목적적 연출이 아니라 '해냈다'는 보람에 복무.
- 정착의 연결감(오늘 배운 것이 나중에 복습으로 다시 온다)을 은은하게.
- 부담·의무감 없이 마무리. 다음 방문을 **강요 아닌 초대**로(죄책감·상실 공포 금지).

# 3. 구성 요소 (구체)
1. **완결 축하 헤더 + 메타포 + 축하 연출(리뷰 반영)** — "오늘 CS 한입 완료!" + 완결감 그래픽 + **눈에 확 들어오는 축하 애니메이션**(예: 컨페티·체크 모션·수치 카운트업) + 햅틱. 성취가 분명히 느껴지게(단 1~2초, 다음 행동을 막지 않게).
2. **오늘의 요약(가볍게)** — 오늘 푼 본 문제 수, (선택) 새로 만난 개념 수 / 복습한 개념 수. 숫자 압박 아닌 성취 표현.
3. **정착 연결 안내(은은)** — "오늘 배운 건 나중에 복습으로 다시 만나요." (간격 반복 암시, 부담 X)
4. **연속 학습(스트릭) — 긍정 동기(기능 6)** — 스트릭 상승을 성취로 또렷이 보여 재접속 동기를 만든다. ⚠️ **끊김에 죄책감·상실 공포 연출 금지**(끊겨도 "다시 시작해요" 톤). 강제 아님.
4-b. **연결 문제 잠금 해제 배지 [반영 2026-07-22: 면접 대비 가치 계획 §3.2 DI7]** — 이번 세션 중 정답 처리로 어떤 연결 문제(다개념 문제)의 마지막 구성 개념이 채워졌으면, 서버 완료 응답의 `unlockedIntegrations` 목록만큼 배지로 보여준다. 문구 예: "TCP·UDP를 모두 배웠어요 — 이제 두 개념을 잇는 문제를 만날 수 있어요"(개념명은 실제 값으로 치환). 완결 축하·스트릭의 성취 위계를 가리지 않는 담백한 톤(작은 배지/카드, 별도 경고색·신규 색 도입 없음 — success 계열의 옅은 톤 재사용). `unlockedIntegrations`가 비어 있으면 표시하지 않는다. 여러 건이면 목록으로 나열.
4-c. **면접 연습 열림 요약 카드 [반영 2026-07-22: DI9]** — 이번 세션에서 스스로 맞혀 새로 면접 연습이 열린 개념이 있으면(서버 응답 `newlyEligibleConcepts`), "오늘 스스로 맞힌 개념 N개의 면접 연습이 열렸어요" 요약 카드를 보여준다. 완결 축하보다 낮은 위계의 담백한 카드(§DESIGN_SYSTEM InterviewUnlockSummaryCard) — 이 화면의 Primary는 여전히 "오늘은 여기까지"이므로, 카드 안에 "면접 연습하러 가기" 정도의 보조 링크(TextLink)만 둘 수 있다. `newlyEligibleConcepts`가 비어 있으면 표시하지 않는다.
   - **배치 순서** — 4(스트릭) 바로 아래 4-b → 4-c 순으로 둔다(다개념 게이트 해제가 먼저, 그다음 면접 열림). 아래 8번 선호 난이도 제안 카드는 그보다 더 아래(가장 하단, 기존 DF1 원칙 그대로)에 유지한다 — 이번 세션에서 "무엇을 해냈는지"를 보여주는 성취 계열(스트릭·잠금 해제·면접 열림)을 먼저 두고, 성격이 다른 설정 제안(난이도 조절)은 맨 마지막에 배치한다.
   - ⚠️ **카드 밀도 주의(시안 단계 검토 과제)** — 스트릭·4-b·4-c·(조건부) 8번까지 겹치면 완료 화면이 카드 3~4장으로 붐빌 수 있다. 실제 시안 단계에서 동시 노출 빈도를 보고 축약(예: 4-b·4-c를 한 카드로 합치기)을 검토한다.
5. **Primary CTA — [오늘은 여기까지]** — 홈으로. 편안한 마무리.
6. **Secondary — [조금 더 풀기]** — 압박 아닌 선택. **[반영 2026-07-19: D6·D9 — 추가 학습 폐지, M2 갱신]** 눌러도 **새 세션**을 시작할 뿐이며 사용자 풀이 경험은 방금 마친 세션과 완전히 동일하다(완료 카운트·완결 화면도 동일하게 받는다). 다만 **스트릭은 그날 첫 세션 완료로만 기록되는 하루 멱등**이므로, 이 세션을 마쳐도 스트릭이 추가로 오르지는 않는다는 점만 버튼 아래 은은한 보조 문구로 표시(예: "스트릭은 이미 오늘 채워졌어요"). 이미 '오늘 완료'로 축하가 끝난 시점이므로, 완료의 흥을 깨지 않는 담백한 톤을 쓴다.
7. **게스트 승계 유도 — 홈으로 일원화(2026-07-16 오너 결정)** — 이 화면에는 두지 않는다. 완료의 순간은 성취에만
   집중하고, 가입 접점은 홈의 가입 유도(GuestUpgradeBanner) 하나로 모은다(접점 분산 시 어디서도 강하게
   전달되지 않는 문제 방지).
8. **선호 난이도 제안 카드(조건부) — [반영 2026-07-21: DF1]** — 도메인 근거(기능 1.5): 온보딩에 난이도 선택을 두지 않는 대신, **선호 난이도를 아직 정하지 않고 이 제안에 응답한 적 없는** 사용자에게 세션 완료 시 **1회만** 제안한다.
   - **노출 조건:** 위 조건을 만족할 때만. 이미 선호를 정했거나 한 번이라도 응답(선택/거절)한 사용자에게는 나오지 않는다. (노출 여부 판단은 서버가 단일 출처로 내려 준다 — 화면은 받은 신호대로 표시.)
   - **배치·위계:** 완결 축하 헤더·축하 연출(구성 1)·스트릭(구성 4)의 성취 위계를 **가리지 않는 하단**에 둔다. 축하가 히어로이고 이 카드는 그 아래 얹히는 **가벼운 초대**다(성취 맥락 보존). **[반영 2026-07-22]** 4-b(잠금 해제 배지)·4-c(면접 열림 요약 카드)보다도 더 아래, **가장 하단**에 둔다(위 4-b·4-c "배치 순서" 참고) — 성취 계열을 다 보여준 다음 마지막에 놓는 설정 제안이다.
   - **문구(무낙인 — 상태 서술형):** 리드는 수준 평가 뉘앙스("실력에 맞는") 없이 조절 주체가 사용자임을 드러낸다(예: "문제 난이도를 맞춰볼까요?"). 선택지는 상태 서술형 3택 — **"CS를 이제 막 시작해요"**(쉬움) / **"기본기를 다지는 중이에요"**(보통) / **"도전적인 문제를 원해요"**(어려움) — 와 거절 **"지금은 괜찮아요"**. 문제 배지의 쉬움/보통/어려움 표기는 별개로 그대로 유지(콘텐츠 속성 표시).
   - **응답 처리:** 셋 중 하나를 고르면 그 선호로 저장하고 카드를 닫는다. **"지금은 괜찮아요"**(거절)를 누르면 "설정에서 언제든 바꿀 수 있어요"를 은은히 안내하고 닫는다. 어느 쪽이든 이후 완료 화면에서 **다시 묻지 않는다**(잔소리 금지 — 기능 6 긍정 동기).
   - ⛔ 수준을 평가·판정하는 뉘앙스(낙인) 금지. ⛔ 축하를 밀어내는 크기·강조 금지.
