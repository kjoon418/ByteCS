**전제:** `00 공통 디자인 원칙.md`. 이 문서는 이 페이지의 목적·구성 요소만 기술한다.

# 1. 어떤 페이지인가
- **로그인·가입 화면** — 게스트가 가입하거나, 기존 사용자가 로그인하는 화면.
- 도메인 근거(기능 4 계정): 게스트로 쌓은 **학습 상태를 가입 시 계정으로 승계**. 인증 수단은 도메인에서 고정하지 않음(구현 판단) → 디자인은 **수단 중립적 틀**로.
- 목적: 마찰을 최소화하면서 "가입하면 기록이 안전해진다"는 가치를 전달.

# 2. 이 페이지의 테마 강조점
- **저마찰·안심.** 입력·단계 최소, "왜 가입하나"를 한 줄로. 부담 주지 않기.
- 승계 안심 메시지가 핵심: "지금까지의 학습 기록이 그대로 옮겨져요."

# 3. 구성 요소 (구체)
1. **헤더 + 가입 이유 한 줄** — "가입하면 학습 기록이 어느 기기에서든 안전하게 이어져요."
2. **인증 수단 버튼(들)** — 소셜/이메일 등(구현에서 확정). **수단이 여럿이어도 시각적으로 정돈**, 인지 부하 최소(UX 1).
3. **이메일 방식 시 입력 폼** — **입력 최적화 키패드**(이메일 패드), 명확한 라벨, 스캐닝 쉬운 배치. 인라인 검증(긍정 문구).
4. **로그인 ↔ 가입 전환** — 같은 화면에서 부드럽게 전환(별도 페이지 남발 X).
5. **Primary CTA — 동사형** — "가입하고 기록 저장하기" / "로그인하기"(엄지 영역 하단).
6. **승계 안내 배너(게스트에서 왔을 때)** — "지금까지 푼 기록이 이 계정으로 옮겨져요." 안심 강조.
7. **에러·실패 처리** — "학습 기록은 안전해요"를 우선 고지 + 해결책(에러 가이드). 빨강 남용 금지. 사용자 언어("시간이 지나 로그아웃됐어요").
8. **약관·개인정보(필요 시)** — 최소·명확. 다크 패턴 동의 강요 금지.

2. 성공 → 이전 맥락으로 복귀(홈 또는 직전 화면), **게스트 학습 상태 승계** 완료 피드백(햅틱).
