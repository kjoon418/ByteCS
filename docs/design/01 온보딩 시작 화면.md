**전제:** `00 공통 디자인 원칙.md`. 이 문서는 이 페이지의 목적·구성 요소만 기술한다.

# 1. 어떤 페이지인가
- **온보딩·시작 화면** — 앱을 처음 연 사용자가 만나는 첫 화면.
- 도메인 근거(기능 4 계정): **가입 없이 게스트로 바로 시작**할 수 있어야 한다. 가입은 강제가 아니라 선택.
- 목적: 3초 안에 "이게 뭔지" 감 잡고 **곧장 문제로** 진입하게 한다(진입 부담 제거 = 존재 이유).

# 2. 이 페이지의 테마 강조점
- **초저마찰.** 읽을 것 최소, 선택 최소. "바로 시작"이 압도적 Primary.
- 서비스 한 줄 약속("5분이면 CS 한입")을 **핵심 메타포 그래픽 하나**로(장식 아님, UX 4).
- 가입 유도는 은은하게(부담 X). "나중에 가입해도 기록이 이어져요" 안심 톤.

# 3. 구성 요소 (구체)
1. **서비스 로고·이름** — CS한입(ByteCS).
2. **한 줄 가치 제안** — "5분이면 CS 한입, 오늘도 가볍게." (대화형·긍정)
3. **핵심 메타포 그래픽 1개** — 서비스 성격을 설명하는 일러스트/아이콘 하나. 의미 없는 장식 배제.
4. **[바로 시작하기] — Primary CTA** — 누르면 **게스트로** 곧장 홈/첫 문제로. 엄지 영역 하단, 가장 강조.
5. **[이미 계정이 있어요 / 로그인] — Secondary** — 05 로그인·가입으로.
6. **가입 안심 문구(작게)** — "가입하지 않아도 시작할 수 있어요. 나중에 가입하면 학습 기록이 그대로 이어져요." (게스트 승계 안내)
7. **(선택) 아주 짧은 소개** — 1~2개 슬라이드/포인트. 스킵 쉽게. 과하면 생략.

# 인터랙션 흐름
1. 앱 첫 실행 → 이 화면.
2. **[바로 시작하기]** → 게스트 세션으로 02 홈(또는 곧장 첫 문제).
3. **[로그인]** → 05.

# 이 페이지 주의
- ⛔ 가입/로그인을 시작의 **전제조건**으로 두지 말 것(게스트 우선).
- ⛔ 긴 튜토리얼로 진입을 막지 말 것(스킵 가능).
