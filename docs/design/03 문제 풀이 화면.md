**전제:** `00 공통 디자인 원칙.md`. 이 문서는 이 페이지의 목적·구성 요소만 기술한다. (이 화면이 서비스의 히어로이므로 가장 상세하다.)

# 1. 어떤 페이지인가
- **문제 풀이 화면** — 일일 세션 중 **문제 하나를 푸는** 화면. 서비스의 히어로.
- 도메인 근거: 기능 1(주관식 단답·결정적 판정·무낙인), 기능 2(단계적 힌트·디딤 문제·파고들기), 기능 2.5(오답 교정 힌트).
- 사용자는 여기서: ①문제를 읽고 ②정답을 **직접 적어** 제출하고 ③막히면 **힌트를 펼치거나** ④**디딤 문제로 파고들었다** 돌아오고 ⑤끝내 안 되면 **정답을 열어** 본다.
- **디딤 문제도 이 화면을 재사용**한다(맥락 배지로 구분).

# 2. 이 페이지의 테마 강조점
- **차분·집중.** 문제 하나에 시선 집중, 여백 넉넉. 코드 스니펫 대응(고정폭).
- **비처벌이 핵심.** 틀림에 빨강·경고·벌점 절대 금지. "아직이에요, 다시 해볼까요?" 톤.
- 파고들기(디딤 문제)는 **'내려갔다 올라오는' 깊이감** 전환으로 표현.

# 3. 구성 요소 (구체)
> **핵심 위계: Primary Action은 '정답 제출' 하나.** 힌트·정답 보기·신고는 secondary로 시각 강도를 낮춘다.

## A. 상단 — 세션·맥락 바
1. **세션 진행 인디케이터** — 오늘의 한입 중 몇 번째 **본 문제**인지(예: `2 / 5`). **시간 카운트다운 아님.** 담백하게.
2. **파고들기 맥락 배지 + 복귀 경로** — 디딤 문제로 내려간 상태일 때만. "〈원래 문제〉를 위한 더 쉬운 문제예요" + **돌아가기**. **여러 겹 가능** → 깊이(스택) 표현 가능해야 함.
3. **나가기/중단** — 재개되므로 **부담 없는 나가기**(경고 모달 지양, "언제든 이어서" 톤).

## B. 문제 영역
4. **문제 질문 텍스트** — 화면의 주인공. 청킹·여백. 코드 스니펫 포맷 대응.
5. **난이도 표시(선택·은은)** — 디딤 문제가 '더 쉬움'을 암시하는 데 도움. 압박 안 되게 최소화.
6. **개념 칩** — ⚠️ **풀기 전에는 숨긴다**(개념명이 정답을 스포일할 수 있음: 정답 '해시 충돌' ↔ 칩 '해시 충돌'). **정답 확인·공개 이후에만** 노출.

## C. 답 입력 영역 (Primary)
7. **주관식 단답 입력 필드** — 텍스트 입력. **입력 최적화 키패드**, **자동 포커스**. 재시도 시 이전 입력 유지·수정 가능.
8. **정답 제출 CTA** — **동사형**("정답 확인하기"). **엄지 영역 하단 고정·확장**. 화면의 유일한 Primary.

## D. 피드백 (상태별 — 무낙인)
9. **정답 상태** — 긍정 피드백(시각 + **햅틱**), "맞았어요!" → 다음 문제로 부드럽게. 이때 개념 칩·간단 해설 노출 가능.
10. **불일치 상태(오답 아님)** — **가볍고 부드러운 인라인 넛지**("아직이에요, 다시 해볼까요?"). **빨강·경고 금지, 토스트로 휙 사라지게 하지 말 것**(설명이 필요하므로). 입력 유지·재시도.
11. **오답 교정 힌트(자동·push)** — 특정 **흔한 오답**을 냈을 때 **요청 없이 자동으로 뜨는 카드/인라인**. 예: "'프로세스'는 …라 이 문제의 답과는 달라요. 다시 도전!" **정답 비노출, 오답 처리 아님**(10보다 내용이 풍부한 카드).

## E. 힌트 (pull · 약→강)
12. **[힌트 보기] 진입점** — secondary. **사용자가 누를 때만**.
13. **단계적 힌트 스텝** — 눌러서 **약→강**으로 하나씩(`L1 키워드 힌팅` → `L2 배경지식 보충`). "더 보기"로 다음 레벨, **이미 연 힌트는 계속 보임**. 레벨 위계가 시각적으로 드러나게.
14. **[더 쉬운 문제로 풀어보기] = 디딤 문제 진입** — 배경지식 보충 힌트 안의 진입점. 누르면 파고들기(이 화면 재사용 + A-2 맥락 배지)로 내려감.

## F. 정답 공개
15. **[정답 보기] 진입점** — **사용자 명시 요청 시에만**. 최후의 도움. 누르면 **모범답안 + 짧은 해설**. 이후 다음 문제로. (도움 신호로 내부 복습에 반영되지만, 화면에 벌점처럼 보이게 하지 말 것.)

## G. 보조·엣지
16. **오류 신고 진입점** — 오버플로우(⋯) 등 **눈에 띄지 않되 접근 가능** → 07 신고 모달.
17. **로딩** — 문제 레이아웃 **스켈레톤**(스피너 지양).
18. **접근성** — 글자 확대 대응, 대체텍스트, 포커스 순서(문제 → 입력 → 제출 → 힌트).

# 인터랙션 흐름
1. 스켈레톤 → 문제. 입력 필드 자동 포커스.
2. 답 제출 → **일치**: 축하 → 다음 문제 / **불일치**: 부드러운 넛지(낙인 X).
3. 흔한 오답이면 **오답 교정 힌트 카드** 자동 표시(정답 비노출).
4. 막히면 **[힌트 보기]** → 약→강 펼침 → 배경지식 힌트에서 **[더 쉬운 문제로]** → 디딤 문제(깊이 전환) → 풀고 **원래 문제로 복귀**.
5. 끝내 안 되면 **[정답 보기]** → 모범답안 → 다음 문제.
6. 세션 마지막 본 문제를 끝내면 **04 세션 완료**로.
