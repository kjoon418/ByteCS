이 문서는, CS한입(ByteCS)의 화면 시안을 디자이너에게 넘기기 위한 **공통 디자인 원칙**이다. 각 페이지 요구사항 문서(`01`~`07`)는 이 문서를 전제로 하고, 페이지 고유의 목적·구성 요소만 따로 기술한다.

> 근거: `docs/도메인 명세서.md`, `docs/ux/UX 핵심 가이드.md`, `docs/ux/UX 에러 응답 가이드.md`.
> 범위: **가벼운 1차 탐색 시안**. 정확한 스펙보다 분위기·구성 요소의 방향을 맞추는 목적. 세부 수치(색값·간격)는 열려 있다.

# 1. 어떤 서비스인가
- **CS한입(ByteCS)** — "5분만 투자해도 CS 공부가 되는" 모바일 학습 앱.
- **대상:** 취업·실무를 준비하는 개발자(지망생).
- **정체성:** 문제 풀이가 히어로, 막혔을 때 **포기 대신 힌트로 넘게** 하는 것이 차별점. 미리·꾸준히 쌓아 정착시키는 학습.
- **플랫폼:** 모바일 우선(Android/Kotlin·Compose), 웹 확장(Compose Multiplatform). **모바일 세로 화면 기준**으로 디자인한다.

# 2. 공통 테마·분위기
한 문장: **"부담 없이 한입, 틀려도 괜찮은, 차분히 집중되는 학습 공간."**

| 축 | 지향 | 근거 |
|---|---|---|
| **부담 없음(bite-sized)** | '시험지'가 아니라 '한입 간식' 같은 가벼움. 작은 목표 하나. | 존재 이유(공부 부담 제거), 1 Page = 1 Action |
| **차분·집중** | 장식 최소, 여백 넉넉, 한 번에 하나에 시선 집중. | UX 3·4 |
| **비처벌·격려** | 실패·오답에 **빨강·경고·벌점 금지**. 격려 톤. | UX 에러 가이드, 무낙인 원칙 |
| **개발자 친화, 딱딱하지 않게** | CS 소재지만 IDE처럼 차갑지 않은 친근한 학습 도구. | 대상 + 대화형 긍정 라이팅(UX 2) |

## 색·톤
- **차분한 뉴트럴 베이스 + 절제된 액센트 1~2색**(진행·성공 등 긍정 신호). 액센트 남용 금지.
- **실패/오답/불일치에 경고색(빨강) 금지.** 상태 위계는 색보다 **명도·아이콘·문구**로.
- **라이트/다크 모드 양쪽 설계**(개발자 선호 + 접근성).

## 타이포·모션
- 스캐닝 효율 우선(축약된 경어체), 본문 가독성 우선. **OS 글자 크기 최대 확대에도 UI가 안 깨지게**(UX 5, 필요 시 바텀시트 전환).
- 모션은 **인지 보조 목적**만: 상태 변화 부드러운 피드백. 의미 없는 장식 애니메이션 배제.

## 라이팅(문구) 규칙
- **긍정·대화형**: "불가능합니다/오류" 대신 "이렇게 하면 돼요" 형태(UX 2·에러 가이드).
- **동사형·결과 예측형 CTA**: "확인/취소" 대신 "정답 확인하기", "가입하고 저장하기"(UX 8).
- **사용자 언어**: 시스템·전문 용어 번역("세션 만료" → "시간이 지나 로그아웃됐어요").

# 3. 공통 컴포넌트·패턴
페이지 전반에서 반복되는 요소. 개별 페이지 문서는 이를 재사용한다.

1. **CTA 위계** — 한 화면에 **Primary Action 하나**만 강조(색·크기). 나머지는 secondary/텍스트 버튼. 주요 CTA는 **엄지 영역(화면 하단) 고정·확장**(UX 6).
2. **상단 바** — 뒤로/나가기 + (해당 시) 맥락 타이틀. 파괴적이지 않은 이동은 경고 모달 없이.
3. **로딩** — 스피너 대신 **콘텐츠 스켈레톤**(UX 7).
4. **빈 상태(Empty State)** — 설명 + 시작 CTA로 **행동 유도형**. 비난·공허 대신 긍정(UX 9).
5. **에러·실패** — 자산(학습 상태)이 안전함을 우선 고지 + 해결/우회 경로. 설명 필요한 내용을 토스트로 휙 지나가게 하지 말 것(에러 가이드).
6. **피드백** — 중요한 완료(정답·가입·저장) 시 시각 + **햅틱**(UX 6).
7. **접근성** — 대체텍스트, 논리적 포커스 순서, 명도 대비, 확대 대응(UX 5).

# 4. 관통 원칙 — 무낙인·사용자 주도
> 서비스는 **막지도 벌하지도 않는다.** 도움(힌트·정답 공개)과 진행은 사용자가 원할 때만. 이 원칙이 모든 화면의 톤을 지배한다.

# 5. 페이지 맵
```
[01 온보딩·시작] --게스트 시작--> [02 홈(오늘의 한입)] --시작/이어하기--> [03 문제 풀이]
      |                                   |                                  | (힌트→디딤 문제로 파고들기: 03 재사용)
      +--가입/로그인--> [05 로그인·가입]    +--세션 끝--> [04 세션 완료] --------+
                                          |
                    [06 계정·설정] <--진입--+          [07 콘텐츠 오류 신고]: 03에서 진입(모달)
```

# 6. 공통 금지 (도메인 위반)
- ⛔ 카운트다운 압박 타이머(세션은 시간이 아니라 **분량** 기반)
- ⛔ 오답 빨강 경고·점수 차감·벌점 연출(무낙인)
- ⛔ 강제 정답 공개(정답 공개는 사용자 요청만)
- ⛔ 죄책감·불안 유발(감정적 다크 패턴, UX 4)
- ⛔ 가입 강제(게스트로 계속 쓸 수 있어야 함 — 가입은 '권유')
