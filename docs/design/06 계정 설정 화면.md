**전제:** `00 공통 디자인 원칙.md`. 이 문서는 이 페이지의 목적·구성 요소만 기술한다.

# 1. 어떤 페이지인가
- **계정·설정 화면** — 사용자가 자기 계정·데이터·앱 설정을 관리하는 화면.
- 도메인 근거(기능 4 계정): 학습 상태는 사용자 소유이며, **로그아웃**·**계정 삭제**(되돌릴 수 없음, 확인 필요)로 통제한다. 게스트는 여기서 **가입 유도**.
- 목적: 데이터 통제권을 명확·안전하게 제공.

# 2. 이 페이지의 테마 강조점
- **명료·신뢰.** 정돈된 목록, 파괴적 동작(삭제)은 신중한 확인.
- 게스트에게는 "가입해서 기록 지키기"를 자연스럽게.

# 3. 구성 요소 (구체)
1. **계정 상태 헤더**
   - 가입자: 프로필(닉네임/이메일 등, 구현 범위) 표시.
   - **게스트: "게스트로 이용 중" + [가입하고 기록 지키기]** 강조(05로). 승계 안내.
2. **학습 관련 설정** — **세션 크기**(제목) / "하루 세션에서 풀 문제 수예요"(보조 설명) / `N문제`(값, 기본 5). −/+ 스텝퍼로 조정, 값을 바꾸면 [저장하기]가 나타난다. 알림 등은 로드맵이므로 MVP는 이 설정 하나로 최소화. (하루 여러 세션·리마인더는 도메인 열린 질문/로드맵 → 지금은 넣지 않아도 됨.)
3. **화면 테마 토글** — 라이트/다크/시스템(공통 다크 대응과 연결).
4. **로그아웃** — 가입자에게. "다시 로그인하면 기록이 그대로예요" 안심 문구.
5. **계정 삭제(파괴적)** — 목록에서 시각적으로 구분(강조하되 위협적이지 않게).
   - 누르면 **확인 단계**: **무엇이 사라지는지 명시**("모든 학습 기록·숙련도·복습 일정이 삭제돼요") + **되돌릴 수 없음** 고지 + 명시적 확인.
   - 문구는 비난 없이 사실 중심(에러/라이팅 가이드). 공용 콘텐츠(문제 등)는 영향 없음은 굳이 강조 불필요.
6. **약관·개인정보·문의 등 링크** — 최소·정돈.
7. **앱 정보/버전** — 하단.

# 인터랙션 흐름
1. 02 홈(또는 내비)에서 진입.
2. 게스트: **[가입하고 기록 지키기]** → 05.
3. 계정 삭제: 확인 단계 → 삭제 → 온보딩(01)로.

# 이 페이지 주의
- ⛔ 삭제를 실수로 누르기 쉽게 두지 말 것(확인 필수), 그러나 빨강 공포 연출도 과하지 않게.
- MVP에 없는 기능(알림·유료 등)을 미리 넣지 말 것(도메인 로드맵/열린 질문).
