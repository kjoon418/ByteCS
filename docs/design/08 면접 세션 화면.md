**전제:** `00 공통 디자인 원칙.md`. 이 문서는 이 페이지의 목적·구성 요소만 기술한다.

# 1. 어떤 페이지인가
- **면접 세션 화면** — 승급된 개념(숙련도 레벨 ≥ 1이 된 개념)에 대해, 면접 방향(이름→설명)의 설명 인출을 AI 루브릭 채점으로 검증하는 **일일 학습 세션과 별개의 활동**. 회원 전용·일일 제한(기본 하루 1세션·3문제).
- 도메인 근거: `docs/plan/면접 대비 가치 구현 계획.md` §3.3(면접 세션)·§3.4(동기 루프 DI9~DI11). 계단 위계(DI1)의 **2단** — 1단(오늘의 한입 주관식)이 진입장벽을 낮추는 히어로라면, 이 화면은 "지금 배운 게 면접에서 유용하겠다"는 믿음을 검증으로 만드는 곳이다.
- 진입은 02 홈의 **면접 연습 진입 카드**에서만 이루어진다(오늘 잔여 쿼터·후보 개념이 있을 때 CTA 활성화). 이 화면 자체는 진입 게이트를 다시 그리지 않는다 — 진입했다는 것은 이미 후보 1개 이상·쿼터 남음을 뜻한다.
- 사용자는 여기서: ①질문을 읽고 ②**자기 말로** 설명을 적어 제출하고 ③채점을 기다리고 ④짚은 포인트·보완할 포인트를 확인하고 ⑤모범 설명을 보고 ⑥다음 질문으로(또는 세션을 마친다).

# 2. 이 페이지의 테마 강조점
- **다른 촉감.** 주관식 단답(짧은 한 줄 정답)과 뚜렷이 구별되는 느낌 — 멀티라인 입력, "내 말로 설명해보세요"라는 서술형 플레이스홀더. 이 화면에 들어왔다는 것 자체가 "한 단계 올라왔다"는 신호가 되게 한다.
- **무낙인 채점.** AI 채점은 합격·불합격·점수를 선고하지 않는다. "짚은 포인트 n개 / 보완하면 좋은 포인트"로만 말한다(§3.3 AI 채점 원칙). 미충족 포인트는 실패가 아니라 다음 재도전의 안내다.
- **명시적 대기.** 채점은 수 초가 걸린다 — 빈 화면·모호한 스피너 대신, "지금 채점 중"임을 분명히 알리는 로딩 상태를 둔다(스켈레톤이 어울리지 않는 드문 예외 — 아직 없는 콘텐츠가 아니라 처리 중인 요청이라서).
- **차분·집중.** 00 공통 원칙 그대로 — 질문 하나, 입력 하나, 결과 하나. 여백 넉넉.

# 3. 구성 요소 (구체)
> **핵심 위계: Primary Action은 단계마다 하나.** 질문 단계 Primary="설명 제출하기", 결과 단계 Primary="다음 질문으로"(마지막 문항이면 "면접 연습 마치기").

## A. 상단 — 세션·맥락 바
1. **세션 진행 인디케이터** — 오늘 면접 세션 중 몇 번째 문항인지(예: `1 / 3`). 03과 같은 분량 기반 표시, 카운트다운 아님.
2. **나가기** — 재개 가능(진행 중 세션은 서버에 영속 — `GET /api/interview/sessions/today`). 부담 없는 나가기, 경고 모달 없이.
3. **오류 신고 진입점** — 오버플로우(⋯). 대상은 **그 면접 질문 콘텐츠**(질문·모범 설명·루브릭). 03의 ReportSheet(§07)와 같은 패턴 재사용.

## B. 질문 단계
4. **질문 텍스트** — 화면의 주인공. `question` 타이포 재사용(03과 동일 스타일 — 다만 정의 재생형 단답 질문과는 문형이 다름: "○○을 설명해보세요" 류).
5. **멀티라인 입력 — ExplanationTextField(신규)** — 주관식 `AnswerTextField`와 **의도적으로 다른 촉감**: 여러 줄 입력, 세로로 늘어나는 텍스트영역, placeholder **"내 말로 설명해보세요"**. 자동 포커스. 최소 글자 수 등 세부 검증은 구현 판단(과도한 차단 금지 — 무낙인).
6. **제출 CTA — [설명 제출하기]** — 동사형, 엄지 영역 하단 고정. 유일한 Primary. 제출 후에는 텍스트영역이 잠기고(더 이상 편집 대상 아님) 곧장 C 단계로 전환된다. **재제출은 없다**(1문항 1채점 — 재도전은 다음 날 새 세션에서).

## C. 채점 로딩 단계
7. **채점 로딩 — GradingLoading(신규, §DESIGN_SYSTEM)** — 제출 직후 명시적 대기 상태. 스피너(`primary`) + "설명을 채점하고 있어요…" 문구. 수 초 내 자동으로 D(결과) 또는 E(폴백)로 전환. 사용자가 끼어들 조작 없음(취소 불가 — 대기가 짧아 허용 오차 안).

## D. 결과 단계 (채점 성공)
8. **루브릭 체크리스트 — RubricChecklistResult(신규, §DESIGN_SYSTEM)** — 포인트별 충족 여부를 나열. 충족 포인트는 체크(✓)+`success` 톤, 미충족 포인트는 중립 아이콘+"보완하면 좋아요" 안내 톤(`neutralNudge` 계열 — **빨강·×표시·danger 금지**). 상단에 집계 한 줄: "짚은 포인트 n개 / 보완하면 좋은 포인트 m개"(**점수·퍼센트·합격선 문구 금지** — §3.3 무낙인 톤). 한 줄 코멘트(AI 응답의 짧은 코멘트) 병기 가능.
9. **모범 설명 — ModelAnswerBlock(재사용, §DESIGN_SYSTEM 5.7)** — 체크리스트 아래, 전체 루브릭 포인트와 함께 공개. 벌점이 아니라 참고 자료 톤(03의 정답 공개와 같은 원칙).
10. **재열람 링크 + 복습 안내(조건부) [반영 2026-07-22: DI10]** — 이번 결과가 **'검증됨'에 못 미칠 때만**(일부 포인트만 충족, 즉 부분/미검증으로 남을 때) 노출:
    - **[그때 푼 문제 다시 보기]** — 그 개념으로 풀었던 주관식 문제의 **읽기 전용 재열람**(스크랩·카테고리 이력 재열람과 동일 패턴 재사용 — 신규 컴포넌트 아님).
    - **"복습에서도 곧 다시 만나요"** — 병기하는 안내 한 줄(DI11 복습 당김 전용 예외 덕분에 사실인 문구). 강제·자동 이동 없음, 담백한 톤.
    - '검증됨'(전 포인트 충족)이면 이 블록 자체를 표시하지 않는다 — 대신 짧은 긍정 한 줄로 충분("이 개념은 면접에서도 잘 설명할 수 있어요" 류, 과장 없이). **[draft — 카피는 구현 시 UX 가이드 기준 확정]** 이 문구는 임시안이다.
11. **다음 CTA** — 기본 **[다음 질문으로]**. 이 문항이 세션의 마지막이면 **[면접 연습 마치기]**로 바뀌며, 누르면 홈으로 복귀(03의 "한입 마치기" 전환과 동일한 사고방식 — 결과 화면을 다 보여준 뒤에 전환).
11-b. **완료 요약 블록(마지막 문항 결과에만) [반영 2026-07-22: 리드 지시 — DI5 스트릭 OR]** — CTA가 [면접 연습 마치기]로 바뀌는 바로 그 결과 화면 하단, 다음 CTA(11) 위에 담백한 완료 요약을 붙인다.
    - **오늘 연습 요약** — "오늘 개념 N개를 면접처럼 설명해봤어요"(N=이 세션에서 채점이 진행된 문항 수).
    - **스트릭(조건부)** — **이번 세션 완료로 스트릭이 실제로 기록된 경우에만** 스트릭 한 줄을 표시한다(완료 응답의 스트릭 필드가 데이터 소스 — 계획 §4.2). 그날 이미 다른 완료(일반 세션 등)로 스트릭이 채워져 있었다면 이 완료는 스트릭을 추가로 올리지 않으므로(DI5 하루 멱등) 스트릭 줄 자체를 표시하지 않는다 — 중복 축하 금지. 04 D6의 "스트릭은 이미 오늘 채워졌어요" 은은한 절제 톤과 같은 원칙.
    - **위계** — 04 세션 완료 화면의 완결 축하(컨페티 등)보다 훨씬 작은 텍스트/카드 블록이다. 이 화면은 04의 큰 축하 연출을 가져오지 않는다 — 완결 축하는 04(오늘의 한입)만의 것이고, 이 블록은 "오늘 면접 연습도 했다"는 담백한 확인일 뿐이다.

## E. 결과 단계 (채점 폴백 — 실패)
12. **폴백 결과 화면 [반영 2026-07-22: §3.3 AI 채점 원칙]** — 채점 실패(타임아웃·오류, 재시도 1회 후에도 실패) 시: 체크리스트 대신 안내 한 줄 **"채점을 잠시 쉬어갈게요 — 모범 설명과 비교해보세요"** + 모범 설명(9)을 그대로 공개. 면접 준비도는 갱신되지 않는다(무낙인 — 사용자 탓 아님을 톤으로 분명히 한다). 이 문항은 세션의 채점 성공 집계에서 제외되며, 세션 전체가 폴백이면 오늘의 쿼터를 차감하지 않는다(화면에는 노출하지 않는 서버 내부 규칙 — 사용자에게는 담백한 안내만).
13. 이 상태에서도 다음 CTA(11)는 동일하게 동작한다 — 폴백이 진행을 막지 않는다.

## F. 보조·엣지
14. **로딩(진입 시)** — 첫 질문을 불러오는 동안은 콘텐츠 스켈레톤(03과 동일 원칙, C의 채점 로딩과는 다른 상황 — "아직 콘텐츠가 없음"이므로 스켈레톤이 맞다).
15. **접근성** — 글자 확대 대응, 대체텍스트, 포커스 순서(질문 → 입력 → 제출 → 결과). 체크리스트는 스크린리더로 "충족"/"보완 필요" 상태를 텍스트로 명확히 낭독(아이콘만으로 전달하지 않음).

# 인터랙션 흐름
1. 02 홈의 면접 연습 카드 → 이 화면. 스켈레톤 → 첫 질문(1/3).
2. 멀티라인 입력에 자기 말로 설명 작성 → **[설명 제출하기]** → 입력 잠김 → 채점 로딩(수 초).
3. 채점 성공 → 결과(체크리스트 + 모범 설명), '검증됨' 미달이면 재열람 링크 + 복습 안내 병기 → **[다음 질문으로]**.
4. 채점 실패 → 폴백 결과(안내 + 모범 설명만, 체크리스트 없음) → **[다음 질문으로]**(동일하게 진행).
5. 마지막 문항의 결과를 다 보여준 뒤, 완료 요약 블록(오늘 연습 개념 수 + 스트릭 기록된 경우만 스트릭 한 줄)이 붙고 CTA가 **[면접 연습 마치기]**로 바뀜 → 누르면 홈으로 복귀.
6. 언제든 **나가기**로 중단 가능 — 다음에 이 화면에 다시 들어오면 진행 중이던 세션이 그대로 이어진다.

# 이 페이지 주의
- ⛔ 점수·퍼센트·합격/불합격 선고 — 체크리스트는 항상 "짚은 포인트 / 보완하면 좋은 포인트" 서술형만.
- ⛔ 미충족 포인트에 빨강·×·경고 아이콘 — 중립 톤(`neutralNudge`)만.
- ⛔ 재제출 허용 — 1문항 1채점, 재도전은 다음 날 세션에서.
- ⛔ 채점 로딩을 무기한 빈 화면·모호한 스피너로 방치 — 반드시 "채점 중" 문구 동반.
- ⛔ 폴백을 에러처럼(ErrorBanner·danger) 다루기 — 담백한 안내 톤 유지, 모범 설명은 그대로 공개.
- ⛔ '검증됨' 개념에도 재열람·복습 안내 블록을 억지로 붙이기 — 그 블록은 미달일 때만.
- ⛔ 완료 요약 블록에서 스트릭이 실제로 기록되지 않았는데도(그날 이미 다른 완료로 채워진 경우) 스트릭을 표시 — 중복 축하 금지, 완료 응답 필드가 없으면 그 줄 자체를 그리지 않는다.
- 게스트는 이 화면에 도달하지 않는다(홈 카드가 가입 유도로 대체) — 이 화면은 진입 게이트를 다시 그리지 않는다.
